Installation#

This section provides instructions on how to install the pytme library. The available options are outlined in the tabs below.

Conda provides an isolated environment, helps avoid conflicts with other packages, and installs necessary dependencies.

conda create \
   --name pytme \
   -c conda-forge \
   python=3.11 \
   pyfftw \
   napari \
   magicgui \
   pyqt

The following will download and install the newest version of pytme into the Conda environment..

conda activate pytme
pip install git+https://github.com/KosinskiLab/pyTME.git

Pip will fetch the required packages from PyPI and install them on your system.

pip install pytme

Note

The Python Package Index (PyPI) provides a simple and convenient way to install pytme. However, installing from Conda or source fetches the most recent version of the code base.

Installing from Source provides you with the latest unreleased changes.

pip install git+https://github.com/KosinskiLab/pyTME.git

Docker provides a consistent and isolated environment for running pytme.

To build the Docker image locally

docker build -t pytme -f docker/Dockerfile_GPU .

Alternatively, you can pull the latest version from Docker Hub

docker pull dquz/pytme:latest

Note

Latest corresponds to the current version of the main branch. You can also use a release version by specifiying the corresponding tag.

CPU/GPU/TPU Support#

pytme’s backend agnostic design enables the same code to be run on practically any hardware platform using a best-of-breed approach. To enable pytme to utilize compute devices other than CPUs, install one of the libraries below. pytme defaults to CuPy for GPU applications, but expert users can decide freely between their desired backend.

The following will install the CuPy dependencies of pytme

pip install "pytme[cupy]"

Alternatively, you can install CuPy directly

pip install cupy-cuda12x

If your CUDA version is lower than 12 or you encounter any issues, please refer to CuPy’s official installation guide for a version tailored to your system and detailed instructions.

The following will install the PyTorch dependencies of pytme

pip install "pytme[torch]"

Alternatively, you can install PyTorch directly

pip install torch torchvision

PyTorch’s installation might vary based on your system and the specific GPU in use. Consult the official PyTorch website for detailed installation options tailored for your environment.

The following will install the JAX dependencies of pytme

pip install "pytme[jax]"

Alternatively, you can install JAX directly

pip install "jax[cuda12]"

Setting up JAX might require additional attention on certain platforms. Consult the JAX documentation for tailored options.

The following will install the MLX dependencies of pytme

pip install mlx

The MLX library is only available for Apple silicon chips.

GUI Setup#

If you would like to perform interactive preprocessing and analysis of your data using the GUI shipped with pytme, you need to install the following dependency

pip install git+https://github.com/maurerv/napari-density-io.git

If you have installed pytme using Conda, thats it. Otherwise, you have to install the remaining dependencies

pip install napari magicgui PyQt5

Testing the Installation#

To verify that pytme has been installed correctly, you can run the test suite provided with the project as follows:

git clone https://github.com/KosinskiLab/pyTME.git
cd pyTME
ulimit -n 4096
pytest

If the tests pass without any errors, pytme has been successfully installed.

Troubleshooting#

The installation of pyFFTW via pip has been troublesome in the past. Consider using the installation method Conda for a smoother experience. Alternatively, pyFFTW can be installed from source. To compile it on my M1 MacBook running homebrew, I had to modify pyFFTW’s setup.py variable self.library_dirs to include the homebrew paths in the EnvironmentSniffer class’s __init__ method as follows:

self.library_dirs = get_library_dirs()
self.library_dirs.extend(["/opt/homebrew/lib", "/opt/homebrew/opt/fftw/lib"]) # Patch
